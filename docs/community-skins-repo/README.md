# claude-pet-skins

Community-contributed palettes for [ClaudePet](https://github.com/loeweErr/ClaudePet). Each skin is a single `palette.json` (10 named color slots) — no PNG art required.

## Browse

| Skin | Author | Preview |
|---|---|---|
| `example` | starter — copy this to make your own | (table is regenerated by the validate.yml workflow) |

Install a skin from this gallery into your local ClaudePet:

```bash
mkdir -p "$HOME/Library/Application Support/ClaudePet/skins"
curl -fsSL https://raw.githubusercontent.com/loeweErr/claude-pet-skins/main/skins/<id>/palette.json \
  | install -m 0644 /dev/stdin "$HOME/Library/Application Support/ClaudePet/skins/<id>/palette.json"
```

Then right-click your cat → **切换皮肤** → pick the new entry.

## Contribute a skin

1. Fork this repo.
2. Pick an id (`my-skin`) — lowercase, hyphens only.
3. Create `skins/my-skin/palette.json` matching the schema below.
4. (Optional) Add `skins/my-skin/preview.png` — a 320×320 PNG showing the cat in the idle pose with your palette.
5. Open a PR. The CI validator runs against your `palette.json`. If green and the look is consistent with the gallery, the maintainer will merge.

## Schema

`palette.json` must validate against [`schema.json`](schema.json). All ten color slots are required, hex format is `#RRGGBB` or `#RRGGBBAA`. Optional `displayName`, `author`, `license` fields are honored by the app.

For per-slot guidance see ClaudePet's own [`docs/creating-skins.md`](https://github.com/loeweErr/ClaudePet/blob/main/docs/creating-skins.md).
