# RFCs

Long-term direction notes. Each RFC is a short doc proposing a major direction — not a commitment to ship. Sequence is exploratory; expect breaking changes to the ideas as discussion happens.

## Format

```
0001-short-slug.md
```

Each RFC has:
- **Status** — Draft / Accepted / Rejected / Superseded
- **Author** — usually the maintainer; community RFCs welcome
- **Summary** — one paragraph
- **Motivation** — why now, why not later
- **Sketch** — rough technical / UX shape
- **Open questions** — what we don't know yet

## Process

To propose a new RFC: open a Discussion in the `Show & Tell` or `Q&A` board first to gauge interest. If there's traction, open a PR adding `docs/rfcs/NNNN-slug.md`. The maintainer marks status after a comment-and-iterate period.
